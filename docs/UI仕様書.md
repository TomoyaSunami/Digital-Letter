
# **デジタルレター（Digital Letter）UI仕様書（v1.0）**

## **0. 前提**
* 対象：Webブラウザ
* チャットアプリではなく、**「手紙を書いて送る」という行為を中心にしたUX」**
* 正常系の基本操作のみを対象（異常系・エラーハンドリングは別途）

---

# **1. 画面一覧（Screen List）**

| No | 画面名                      | 目的                 |
| -- | ------------------------ | ------------------ |
| 1  | **オンボーディング**             | サービスの世界観伝達・初回導線    |
| 2  | **ログイン / 新規登録**          | メール or SNSでアカウント作成 |
| 3  | **ホーム / 手紙箱**            | 受信手紙・送信済み手紙の一覧     |
| 4  | **手紙詳細（閲覧）**             | 封筒開封 → 手紙を見る体験     |
| 5  | **新規手紙作成（Editor）**       | 手紙内容の入力／装飾         |
| 6  | **便箋・封筒選択**              | テーマ・スタイル選択         |
| 7  | **送信設定（遅延時間 / タイムカプセル）** | 配達時間を決定            |
| 8  | **送信完了（配達中画面）**          | “配達中アニメーション”表示     |
| 9  | **プロフィール設定**             | 名前・アイコン・テーマ設定      |
| 10 | **設定（Settings）**         | アカウント、通知、保存容量      |

---

# **2. 画面仕様（正常系）**

---

## **2.1 オンボーディング**

### ● 目的

世界観の提示 / アプリの核心機能紹介

### ● UI構成

* フルスクリーン画像（手紙・封筒のアニメーション）
* キャッチコピー
  　例：「デジタルで、ていねいな文通を。」
* 機能紹介スライド（3枚）
  　1. “ゆっくり届く”手紙
  　2. 手紙箱で思い出が積もる
  　3. タイムカプセル

### ● ボタン

* 「はじめる」 → ログイン画面へ

---

## **2.2 ログイン / 新規登録**

### ● UI構成

* メールアドレス入力
* パスワード入力
* SNSログイン（Google / Apple）
* 「新規登録」「ログイン」

---

## **2.3 ホーム / 手紙箱**

### ● 概要

受信・送信手紙の一覧。アプリの中心。

### ● UI構成

**上部：**

* アイコン（プロフィール）
* 右上：＋ボタン（新規手紙作成）

**メインエリア：**
タブ構成

* 「受信箱」
* 「送信箱」

リスト表示（カード型）：

* 封筒のサムネイル
* 相手の名前
* 未開封マーク
* 送信日時 / 配達予定時刻
* 開封済みの場合は “開封日” を表示

### ● アクション

* 封筒タップ → 手紙詳細へ

---

## **2.4 手紙詳細（閲覧）**

### ● ステップ構成

1. **封筒のアニメーション**
   　・スワイプで開封
   　・効果音（紙の音）

2. **手紙ビュー**
   　- スクロール可能
   　- 手書き風フォント
   　- 写真・音声・動画を差し込み表示
   　- 背景は選択した便箋

3. **フッター**
   　- 「返信を書く」
   　- 「保存（お気に入り）」
   　- 「共有（スクショ / PDF）」※MVPでは省略可

---

## **2.5 新規手紙作成（Editor）**

### ● UI構成

**ヘッダー：**

* 戻る
* 下書き保存
* 次へ（送信設定へ）

**手紙エリア：**

* 手書き風入力フィールド（高さ自動拡張）
* プレースホルダ：「ここに手紙を書いてください」

**下部ツールバー：**

* 便箋選択
* 写真添付
* 音声添付
* 動画添付
* スタンプ（シール）

---

## **2.6 便箋・封筒選択**

### ● UI構成

* グリッド表示（テンプレ一覧）
* プレビュー
* 選択 → ハイライト
* 有料テーマには「Premium」タグ

---

## **2.7 送信設定（遅延時間 / タイムカプセル）**

### ● UI構成

* 「配達までの時間」スライダー
  　（例：3分／1時間／12時間／24時間／1週間／1年）
* カレンダー選択（10年後まで設定可）
* メッセージ：「◯◯さんには、●●に届きます」

**ボタン：**

* 「送信する」

---

## **2.8 送信完了（配達中画面）**

### ● UI構成

* 配達アニメーション（鳥／郵便車／星空など）
* テキスト：「◯時間後に届きます」
* 「ホームへ戻る」ボタン

---

## **2.9 プロフィール設定**

### ● UI構成

* 名前
* アイコン（画像アップロード）
* ひとこと（任意）
* テーマカラー（アプリのアクセント色切替）

---

## **2.10 アプリ設定**

### ● UI構成

* アカウント情報
* 通知設定（配達通知）
* 保存容量
* プレミアムプラン
* 利用規約・プライバシー

---

# **4. UIコンポーネント一覧**

| コンポーネント     | 説明             |
| ----------- | -------------- |
| 手紙カード       | 封筒プレビュー＋送受信情報  |
| 封筒開封アニメ     | スワイプ動作＋紙の音     |
| 手紙ビュー       | 背景（便箋）＋本文スクロール |
| 添付ビュー       | 写真/動画/音声など     |
| 便箋テンプレ一覧    | グリッドUI         |
| 遅延時間スライダー   | 配達までの時間指定      |
| タイムカプセル日付選択 | カレンダーUI        |
| 手紙箱フィルタ     | 受信/送信タブ        |

# **5. MVPに含むUI**

MVPとして必須のUIのみ抽出：

* ホーム（手紙箱）
* 新規手紙作成
* 送信設定（遅延設定）
* 手紙詳細（開封）
* ログイン/新規登録
* 配達中画面

---

必要であればさらに以下も作成できます：

✔ **ワイヤーフレーム（白黒）**
✔ **画面デザイン案（カラー案）**
✔ **コンポーネント仕様書（API連携含む）**
